## 🚀 下一步操作

### 立即开始（5 分钟）

1. **安装 Hugo Extended**
   ```bash
   # Windows
   choco install hugo-extended -y
   
   # macOS
   brew install hugo
   ```

2. **初始化项目**
   ```bash
   cd d:\qoder\blog
   hugo mod init blog
   hugo mod get
   ```

3. **本地预览**
   ```bash
   hugo server -D
   ```
   访问 http://localhost:1313

4. **修改个人信息**
   - 编辑 `hugo.toml`
   - 修改 baseURL、title、author 等

5. **部署到 Cloudflare Pages**
   - 参考 `QUICKSTART.md`

## 🔧 需要修改的配置

在开始使用前，请修改以下文件：

### 1. hugo.toml（必须修改）
```toml
baseURL = 'https://yourdomain.com/'  # 改为你的域名
title = '我的个人博客'                # 改为你的博客名

[params]
  author = "Your Name"                # 改为你的名字
  description = "博客描述"            # 改为你的描述
  
[[params.socialIcons]]
  name = "github"
  url = "https://github.com/yourusername"  # 改为你的 GitHub
```

### 2. content/about.md（建议修改）
- 修改个人介绍
- 添加技能栈
- 更新联系方式

### 3. static/favicon.ico（建议替换）

## 📦 部署方式总结

### 方式一：Git 集成（推荐）⭐
**优点**: 最简单、自动化
**流程**:
1. 推送代码到 GitHub
2. 在 Cloudflare Pages 连接仓库
3. 配置构建设置
4. 自动部署

**详细步骤**: 查看 `QUICKSTART.md` 或 `USAGE.md`

### 方式二：GitHub Actions
**优点**: 更多控制、可自定义
**流程**:
1. 设置 GitHub Secrets
2. 推送代码触发 Actions
3. 自动构建并部署

**配置文件**: `.github/workflows/deploy.yml`

### 方式三：Wrangler CLI
**优点**: 本地直接部署、无需 Git
**流程**:
1. 安装 Wrangler
2. 运行 `hugo --minify`
3. 执行 `wrangler pages deploy`

**配置文件**: `wrangler.toml`
